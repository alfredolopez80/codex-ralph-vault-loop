# Exit Review

## Purpose

Close a task or migration phase with clear evidence, no leaked sensitive content, and useful sanitized memory.

## Review Checklist

Confirm that the scope matches the phase request and that the previous checkpoint requirement was respected. Check that changed files are limited to the intended surface, validations ran, risks are explicit, and PASS or FAIL is justified. No secrets, raw vault content, or RED material may be written.

## Learning Capture

Classify learnings before saving them. GREEN learnings are reusable and safe for global memory. YELLOW learnings are project-specific and sanitized. RED learnings are not saved.

Write only GREEN/YELLOW summaries to `~/.ralph-codex` or the vault when useful.

## Handoff Shape

The handoff should name changed files, validation evidence, decisions, unresolved risks, and next phase readiness.

Do not include raw command dumps unless the exact output is necessary.
