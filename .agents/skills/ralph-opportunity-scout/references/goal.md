# Ralph Goal Scout

Use this reference when a task wants sustained execution but lacks a clean,
checkable finish line.

## Action Policy

Propose `$ralph-objective-prep` before native `/goal` for broad, risky,
ambiguous, recovery-oriented, audit-oriented, or plan-driven goals. Do not set
native goals or create goal boards without approval.

For simple concrete goals, defer to native goal handling directly when the
runtime supports it.

## Recognition Signals

- The user asks to keep going, drive something to done, recover a broken state,
  or pursue a broad objective autonomously.
- Success depends on discoveries made along the way.
- The work spans multiple turns, tools, or validation stages.
- The stated outcome could drift or succeed while solving the wrong problem.
- Completion evidence is not obvious.

## Ralph Fit

Use `$ralph-objective-prep` to produce:

- interpreted outcome;
- non-goals and constraints;
- proof of completion;
- likely blind spots;
- first safe task;
- whether a native `/goal` should be set.

Then propose the native goal only when the objective is short, verifiable, and
not competing with another active goal.

## Stay Inline When

- The task can be completed this turn.
- The steps and proof are already obvious.
- The user is actively steering each step and does not want autonomy.
- Goal persistence is unavailable and a conversational fallback is enough.

## Proposal Template

```text
Opportunity: This is goal-shaped and could drift without explicit proof.
Best Ralph path: Use `$ralph-objective-prep`, then native `/goal` once scoped.
Why now: <ambiguous finish line or multi-turn risk>.
Approval needed: Yes, before creating a goal board or setting a native goal.
Inline fallback: I will ask one clarifying question or run a bounded first pass.
```
