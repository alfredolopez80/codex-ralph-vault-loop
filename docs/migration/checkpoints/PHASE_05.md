## Scope

This phase adds deterministic local vault scripts under `scripts/vault`. The default vault path is `~/Documents/Obsidian/MiVault`, but tests and manual checks use `VAULT_DIR` so no private vault data is touched.

## Implementation

`vault-init.py` creates the required global, project, agent, and template directories. `vault-save.py` writes GREEN notes to `global/raw`, writes YELLOW notes to the active project `raw` folder, deduplicates by SHA-256 hash, and skips RED content without persistence.

`vault-search.py` provides pure-Python text search. `vault-index.py` writes a JSON index from Markdown frontmatter. `vault-compile.py` builds a project handoff file. `vault-demote.py` moves a note back to project raw storage. `vault-lint.py` checks basic frontmatter and simple RED markers.

## Validation Results

The manual flow was run with a temporary `VAULT_DIR`: init created the structure, GREEN save persisted a note, RED save skipped persistence, and search found the GREEN note. Index, lint, and compile also ran successfully against a temporary vault. The integration test `tests/integration/test_vault_basic.py` passed with pytest.

Python syntax checks passed for the vault scripts. Secret scans over `scripts/vault`, `tests/integration`, and this checkpoint returned no findings. Direct provider scans found no Z.ai or MiniMax `model_provider` configuration.
