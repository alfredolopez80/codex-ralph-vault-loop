Memory is useful only after identity, safety, and scope agree.

Ralph Memory separates the stable code checkout from the active Codex worktree, writes runtime memory under a project id, treats MiVault inbox as quarantine, and lets ralph-recall read only scoped safe memory by default.

Executable root

RALPH_CODE_ROOT

Global hooks resolve Ralph scripts from the stable checkout recorded in ~/.codex/hooks/.ralph-repo-root. This root tells Codex where the code lives, not which project is being worked on.

Active workspace

ACTIVE_WORKSPACE_ROOT

The active project comes from hook payload cwd, tool workdir, PWD, and git metadata. ActiveContext turns that into project_slug, project_id, branch, sha, session id, and workspace instance id.

Safety

RED never persists or injects

Checkpoint, handoff, dream, inbox review, and recall all classify or sanitize before storing or printing useful context.

Identity

Project, session, workspace

Rehydration requires matching project_id, session or continuation rules, workspace instance, TTL, and compatible branch metadata.

Budget

15% handoff reinjection

If the sanitized handoff body fits the configured context ratio, wakeup injects it compactly. Oversized content is deterministically compacted or skipped by the hard cap.

Graduation model

Aristotelian review is a schema, not decoration.

Each review decision records the candidate hash, source project metadata, classification, confidence, decision, target, and an aristotle rationale block. The decision chooses the smallest safe action.

Inbox rule

Inbox is quarantine

Inbox/raw files are not read by default. They appear only when a diagnostic recall explicitly asks for --include-raw.
